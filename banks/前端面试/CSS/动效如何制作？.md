前端做动效有四类方式，按场景选；**性能要点是面试重点**。

## A. 纯 CSS（能用就优先，性能最好，不占 JS 主线程）

- **`transition`（过渡）**：状态 A→B 平滑过渡，最常用（hover 变色、展开收起）。
  ```css
  .box { transition: transform .3s ease; }
  .box:hover { transform: scale(1.1); }
  ```
- **`animation` + `@keyframes`（关键帧）**：多阶段、可循环、自动播放的复杂动画（loading 转圈）。
  ```css
  @keyframes spin { to { transform: rotate(360deg); } }
  .loader { animation: spin 1s linear infinite; }
  ```

## B. JS 驱动（需要逻辑/实时计算时）

- **`requestAnimationFrame`（rAF）**：JS 做流畅动画的**正确姿势**，浏览器每次重绘前回调（约 60fps）。比 `setInterval` 好：跟屏幕刷新同步不丢帧，页面切后台自动暂停省电。
- **Web Animations API**：`el.animate(...)`，JS 版 keyframes，兼顾性能与控制力。
- 适合跟手拖拽、物理效果等需实时计算，或要精细控制播放/暂停/进度的场景。

## C. 复杂图形 / 逐帧

- **Canvas**：逐帧手绘像素，适合大量元素（粒子、游戏、可视化）。位图、不可选中、无 DOM 语义。
- **SVG**：矢量、可缩放不糊，适合图标、路径动画。
- **GIF / 视频 / Lottie**：现成素材。GIF 体积大、色彩差；复杂动画现多用 Lottie（AE 导出 JSON，体积小可控）。

## D. 🔑 性能要点（核心）

做动画尽量**只改 `transform` 和 `opacity`**：

| 改什么 | 触发什么 | 代价 |
| --- | --- | --- |
| `width`/`height`/`top`/`left`/`margin` | **重排 Reflow**（重算布局）+ 重绘 | 最贵，易卡 |
| `color`/`background` | **重绘 Repaint** | 中等 |
| **`transform`/`opacity`** | **只走合成 Composite**（GPU） | **最便宜，丝滑** |

渲染管线：**JS → 样式计算 → 布局(Layout/重排) → 绘制(Paint/重绘) → 合成(Composite)**。

- 改 `left`：从**布局**这步开始全部重来（最贵）。
- 改 `transform`/`opacity`：**只动最后的合成**，交给 GPU 单独挪图层，布局/像素都不变，故能稳 60fps。

记忆钩子：动画**别碰** `left/top/width`（重排），用 `transform: translate/scale` 让 GPU 单独合成。
