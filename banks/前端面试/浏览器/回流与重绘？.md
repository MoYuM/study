回流和重绘本质上是浏览器渲染页面的步骤之一

浏览器绘制页面的步骤如下：

1. 构建 DOM 树
2. 构建 CSSOM 树
3. 构建渲染树
4. 布局：计算元素的位置、大小
5. 绘制：计算元素的颜色等
6. 合成

- 回流（Reflow）：就是重新跑一遍「布局」，改变几何（尺寸/位置）触发重新布局，开销大。
- 重绘（Repaint）：就是重新跑一遍「绘制」，改变外观但不改布局。
- transform / opacity 是仅有的两个 compositor-only 属性：当元素**已被提升为独立合成层**时，对其做动画可跳过 layout/paint；若未提升仍可能触发 paint。

优化：批量改、读写分离避免强制同步布局、用 class 批量改样式、用 DocumentFragment。

## 追问

### 什么是强制同步布局？为什么危险？

浏览器为了性能会把多次 DOM 修改**批量攒起来**，等空闲时一次性计算布局。但如果你在**写完 DOM 之后立刻读取布局属性**（如 `offsetWidth`、`offsetHeight`、`getBoundingClientRect`、`scrollTop` 等），浏览器被迫立刻刷新队列、重新计算布局才能返回准确的值，这就是**强制同步布局（Forced Synchronous Layout）**。

```js
// 危险：循环里读写交替，每次循环触发一次 reflow
for (let i = 0; i < items.length; i++) {
    items[i].style.width = container.offsetWidth + 'px'; // 读 offsetWidth → 强制 reflow
}

// 正确：读写分离，只触发一次 reflow
const width = container.offsetWidth; // 统一先读
for (let i = 0; i < items.length; i++) {
    items[i].style.width = width + 'px'; // 只写
}
```

会触发强制同步布局的常见属性：`offsetWidth/Height`、`clientWidth/Height`、`scrollTop/Left`、`getBoundingClientRect()`、`getComputedStyle()`。
