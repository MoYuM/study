深拷贝递归复制对象所有层级，拷贝后与原对象完全独立。

## 三个层次（先分清）

- `let b = a`：同一个对象，啥都没复制，改 `b` 连 `a` 一起改。
- **浅拷贝**（`{...a}` / `Object.assign`）：第一层是新的，嵌套的对象/数组仍共享引用。
- **深拷贝**：每一层都重新创建，两者彻底独立。

## 三种方案对比

| 方案 | 循环引用 | 函数 | undefined/Symbol | 特殊类型(Date/Map/Set) |
| --- | --- | --- | --- | --- |
| `JSON.parse(JSON.stringify(x))` | ❌ 直接抛错 | ❌ 丢失 | ❌ 丢失 | ❌ Date 变字符串 |
| `structuredClone(x)`（原生） | ✅ | ❌ 报错 | ✅ | ✅ |
| 手写递归 | ✅（WeakMap） | 看实现 | ✅ | 需自己处理 |

`JSON` 那套是「穷人版」，简单但坑多；`structuredClone` 是原生正规军，唯一短板是不支持函数和 DOM 节点。

## 两个高频坑

1. **`JSON.stringify` 对函数/undefined/Symbol 是「直接丢弃」，不是变字符串**：
   ```js
   JSON.stringify({ a: 1, fn: () => {}, u: undefined }) // '{"a":1}' 键整个消失
   JSON.stringify([1, () => {}, undefined])             // '[1,null,null]' 数组里就地变 null
   ```
   （`[object Function]` 那种字符串是 `String(fn)`/`toString` 的产物，与 `JSON.stringify` 无关。）
2. **`typeof null === 'object'`**：手写判断要先把 `null` 和基本类型一起挡掉。

## 手写要点 + 极简实现

基本类型直接返回；用 WeakMap 记录已拷贝对象解决循环引用；区分数组与对象；Date / RegExp / Map / Set 等特殊类型单独处理；函数通常直接引用。

```js
function deepClone(obj, map = new WeakMap()) {
  if (obj === null || typeof obj !== 'object') return obj; // 基本类型 + null
  if (map.has(obj)) return map.get(obj);                   // 循环引用：取备忘录
  const copy = Array.isArray(obj) ? [] : {};
  map.set(obj, copy);                                      // 先记账，再递归
  for (const key of Reflect.ownKeys(obj)) copy[key] = deepClone(obj[key], map);
  return copy;
}
```

**为什么用 WeakMap 而非 Map**：循环引用（如 `a.self = a`）会让递归无限下钻爆栈，用一张「原对象→新副本」的备忘录在重复碰到时直接取已建副本即可；WeakMap 对 key 是弱引用，拷完原对象能正常被 GC，不会内存泄漏。
