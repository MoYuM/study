对象通过 `__proto__` 指向构造函数的 `prototype`，逐级向上查找属性/方法，形成原型链。

## 三个关键名词（务必分清）

| 名词 | 挂在谁身上 | 指向谁 |
| --- | --- | --- |
| `prototype` | **函数**才有 | 用它 new 出来的实例的原型对象 |
| `__proto__` | **每个对象**都有 | 自己的上级原型 |
| `constructor` | 在 prototype 对象上 | 指回构造函数本身 |

```js
function Person() {}
const p = new Person();
p.__proto__ === Person.prototype          // ✅
Person.prototype.constructor === Person   // ✅
```

记忆钩子：`prototype` 是函数发给后代的「家谱」，`__proto__` 是每个对象攥着的「上一辈地址」，`constructor` 是家谱上写的「祖宗是谁」。

## 查找过程与顶层

访问 `p.toString()`：自己没有 → 顺 `p.__proto__`（=`Person.prototype`）找 → 再顺 `Person.prototype.__proto__`（=`Object.prototype`）找到 → 若到 `Object.prototype.__proto__` 仍没有，则为 **`null`**，返回 `undefined`。

顶层是 `Object.prototype`，其 `__proto__` 为 `null`。

## 核心坑：prototype 对象 vs 函数本身

```js
Person.__proto__           === Function.prototype  // 因为 Person 是个"函数"
Person.prototype.__proto__ === Object.prototype    // 因为 Person.prototype 是个"普通对象"
```

`xxx.prototype` 永远是个普通对象，其链尽头经过 `Object.prototype` 到 `null`；只有问「函数自身的 `__proto__`」时才轮到 `Function.prototype`，而它最终也通向 `Object.prototype`——**条条大路通 `Object.prototype`**。实例 `p` 走的是 `Person.prototype` 这条线，与 `Function.prototype` 无关：

```
p  →  Person.prototype  →  Object.prototype  →  null
```

## 相关考点

- **`instanceof`**：本质是看「`B.prototype` 在不在 `a` 的原型链上」，挨个 `__proto__` 往上比对。
- **`Object.create(null)`**：造一个无原型对象，连 `toString`/`hasOwnProperty` 都没有，常用作纯净字典。
- **`hasOwnProperty`**：判断属性是对象自己的还是从原型链继承的（`in` 会连原型链一起算）。
