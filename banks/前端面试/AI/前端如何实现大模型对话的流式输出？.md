两种主流方式：

## ① SSE / EventSource

后端按 `data: <chunk>\n\n` 格式推送，前端监听 `onmessage` 逐块拼接、增量更新 state：

```js
const es = new EventSource('/api/chat')
es.onmessage = (e) => {
  if (e.data === '[DONE]') { es.close(); return }
  setText(prev => prev + JSON.parse(e.data).content)
}
```

缺点：只支持 GET、不能自定义请求头（跨域受限）。

## ② fetch + ReadableStream

现代 AI SDK 主流方案（OpenAI SDK、Vercel AI SDK 等），支持 POST 和复杂请求体：

```js
const res = await fetch('/api/chat', { method: 'POST', body: JSON.stringify(prompt) })
const reader = res.body.getReader()
const decoder = new TextDecoder()
while (true) {
  const { done, value } = await reader.read()
  if (done) break
  setText(prev => prev + decoder.decode(value))
}
```

优点：支持 POST、可带复杂请求体和自定义 header。

## 其他要点

- 批量渲染避免抖动（`requestAnimationFrame` / 时间片）
- 自动滚底但不干扰用户上滑
- `AbortController` 支持中止
- Markdown 流式截断问题：需对未闭合的标签做自动补全
