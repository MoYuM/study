## 一、请求层：拿到流

两种主流方式：

### ① SSE / EventSource

后端按 `data: <chunk>\n\n` 格式推送，前端监听 `onmessage` 逐块拼接：

```js
const es = new EventSource('/api/chat')
es.onmessage = (e) => {
  if (e.data === '[DONE]') { es.close(); return }
  setText(prev => prev + JSON.parse(e.data).content)
}
```

适合：后端主导推送、不需要传复杂参数的场景。  
缺点：只支持 GET、不能自定义请求头（无法带 Authorization token）、跨域受限。  
优点：浏览器内置自动重连。

### ② fetch + ReadableStream

现代 AI 对话主流方案（OpenAI SDK、Vercel AI SDK 等）：

```js
const res = await fetch('/api/chat', {
  method: 'POST',
  headers: { 'Authorization': `Bearer ${token}` },
  body: JSON.stringify({ messages }),
  signal: controller.signal,   // 支持中止
})
const reader = res.body.getReader()
const decoder = new TextDecoder()
while (true) {
  const { done, value } = await reader.read()
  if (done) break
  setText(prev => prev + decoder.decode(value))
}
```

优点：支持 POST、自定义 Header、可中止（AbortController）。  
缺点：自动重连需手动实现。

> 本质：两种方式都是服务端边生成边 flush（`Transfer-Encoding: chunked`），前端边收边处理。

## 二、渲染层：把流显示好

### 1. 中止请求

用 `AbortController` 让用户随时停止生成：

```js
const controller = new AbortController()
// 用户点「停止」时：
controller.abort()
```

### 2. 批量渲染，避免抖动

逐 token setState 会触发过多重渲染，导致页面抖动。用 `requestAnimationFrame` 或定时器攒批：

```js
let buffer = ''
// 每帧或每 16ms 统一 flush 一次
requestAnimationFrame(() => {
  setText(prev => prev + buffer)
  buffer = ''
})
```

### 3. 自动滚底 + 不打断用户上滑

用户正在上滑查看历史时，不能强制拉到底：

```js
const isAtBottom = () =>
  container.scrollHeight - container.scrollTop - container.clientHeight < 50

// 每次新 chunk 来时
if (isAtBottom()) container.scrollTop = container.scrollHeight
```

### 4. Markdown 流式截断补全

流还没结束时，文本可能处于未闭合状态（如 `\`\`\`` 代码块只有开没有闭），直接渲染会布局崩掉。

解法：渲染前对未闭合的 Markdown 语法做自动补全（补 `\`\`\``、`**`、`_` 等），或用支持增量渲染的 Markdown 库（如 `marked` streaming 模式）。

### 5. 错误与超时处理

```js
try {
  // ...读流...
} catch (e) {
  if (e.name === 'AbortError') return   // 用户主动中止，忽略
  setError('网络异常，请重试')
}
```
