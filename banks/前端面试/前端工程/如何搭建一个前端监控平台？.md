三大模块：**采集 → 上报 → 存储/处理/告警**

## 采集

| 类型 | 采集方式 |
|------|---------|
| JS 错误 | `window.onerror`（同步错误）、`unhandledrejection`（Promise 未捕获）、框架 `ErrorBoundary` |
| 资源错误 | `addEventListener('error', handler, true)`（捕获阶段，img/script 加载失败） |
| 性能 | `PerformanceObserver` + Web Vitals（LCP / CLS / INP） |
| 行为埋点 | 点击事件、曝光（`IntersectionObserver`）、自定义业务埋点 |

## 上报方式对比

| 方式 | 跨域 | 页面关闭后 | 特点 |
|------|------|-----------|------|
| `sendBeacon` | ✅ | ✅ **仍可上报** | 浏览器异步发送，**POST**，不阻塞页面，专为卸载时上报设计 |
| 图片打点 `new Image().src=url` | ✅ | ❌ 会中断 | 最简单，GET，数据量受 URL 长度限制 |
| `fetch` | ✅ | ❌ 会中断 | 灵活，但页面关闭时请求被取消 |

> beacon 优先：`visibilitychange` / `beforeunload` 时机的上报必须用 beacon，它是唯一能可靠发出的方式。beacon 支持 POST（不是 GET），也支持跨域。

**上报最佳实践：** 批量上报 + 采样（不是每条都报，降低服务器压力）+ 失败重试

## 后端处理

- **SourceMap 还原**：生产环境代码压缩后堆栈是 `bundle.min.js:1:123456`，必须用 SourceMap 映射回源码行列
- **聚合去重**：相同错误合并计数，避免告警风暴
- **告警阈值**：错误率/PV 超阈值触发通知
- **看板**：错误趋势、设备/版本分布、用户影响范围

## 核心原则

低侵入、不影响主流程性能、数据可定位到用户与版本。
