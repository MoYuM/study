# 掌握度陪练（Mastery Drill）

## 这个技能要解决的核心问题

学习者最大的盲区是「能力自评失真」：看过答案、标记「已读/已背」就以为掌握了，但一被追问就露馅。真正的掌握不是「看过」，而是**能脱稿讲清「是什么/为什么/什么场景/边界与坑」，并接得住追问**。

本技能把抽象的「我掌握了吗」变成一个可量化、可定位、可持续的训练闭环。核心是三件被学习科学反复验证的事：**主动回忆**（先盖答案自己产出，而不是重读）、**针对性追问**（暴露伪掌握）、**间隔复习**（对抗遗忘曲线）。请始终围绕这三点设计每一次互动。

## 第一步：确定数据源与跟踪方式

不同用户的材料存放方式不同，先判断用哪种，再开始。**不要假设，先快速确认。**

1. **本地题库（默认，题库与进度分离）**：题目内容和学习进度分开存——
   - **题目内容**：`banks/<题库名>/<分类>/<题目>.md`，frontmatter **只存内容**（题目/分类/id + 该题库 `meta.yml` 声明的可选字段，如 频率），正文存答案。每个题库一个 `meta.yml` 声明分类清单与可选字段。
   - **学习进度**：`progress/<题库名>/<id>.md`，frontmatter 存全量数据（题目/分类/频率/id + 掌握/下次复习/上次评测/备注）。未评测的题 `掌握: 🆕未评测`，评过的题有具体档位。同时作为 Obsidian Database 的数据源，用 `前端面试进度.base` 可视化查看。

   ```markdown
   # banks/前端面试/JS/深拷贝.md —— 纯内容，绝不含进度字段
   ---
   题目: 深拷贝
   分类: JS
   频率: 高频
   id: 55e72a4d-...
   ---
   深拷贝递归复制对象所有层级……（答案正文）
   ```
   ```markdown
   # progress/前端面试/55e72a4d-....md —— 进度 + 元信息
   ---
   题目: 深拷贝
   分类: JS
   频率: 高频
   id: 55e72a4d-...
   掌握: 🔴生疏
   下次复习: 2026-06-25
   上次评测: 2026-06-24
   备注: 循环引用(WeakMap)忘了
   ---
   ```

   陪练：`pick.py` 读 `banks/`（题目列表）+ `progress/`（进度），揭晓读 bank 正文，评完**只写 `progress/<题库>/<id>.md` 的 frontmatter**，不碰 banks/ 内容。

确认走 **`AskUserQuestion` tool**（不要用自由正文问），例如 `question`=「练哪一份？」，`options`=本地题库目录 / 给我一份题目文件 / 报个主题我来出题。

## 辅助脚本（本地题库自带，先用脚本别手动汇总）

题库根目录有现成脚本，覆盖陪练里最高频的两件事——**看现状**和**挑题**。优先调用它们。

- **看当前状态**：`python3 stats.py [题库名]`。输出各题库的总题数 / 未评测 / 🔴🟡🟢⭐ 分布 / 今日到期数 / 分类分布，外加合计；给个题库名只看那一份。用户问「现在练到哪了 / 题库啥情况 / 还剩多少没练 / 哪类最弱」时直接跑它，别自己翻文件。
- **快速出题**：`python3 pick.py [题库名] [筛选...]`。按优先级（到期＞未评测＞低档位＞高频）**盖答案**选题，每题输出 档位 / 到期 / 频率 / 分类 / 题目 / id / 文件路径 / **上次备注**——**但绝不含答案正文**。常用筛选：
  - `-n 8` 出几题（默认 5）
  - `--due` 今天到期复习　`--new` 从未评测　`--weak` 只挑 🔴🟡
  - `--cat JS,React` 按分类　`--field 频率=高频` 按可选字段　`--level 🟢` 按档位
  - `--random` 纯随机　`--json` 结构化（要精确解析时用）　`--all` 不截断（看池子多大）
  - 例：`python3 pick.py --due`（练今天该复习的）、`python3 pick.py --new --cat AI --field 频率=高频`（AI 高频里挑没练过的）

  输出里的 **`file` 用来在第 5 步揭晓时读答案正文，`id` 用来在第 6 步写回 `obsidian-db`**；**`上次备注`** 记着这道题上回栽在哪，正好拿来设计追问。脚本只负责选题，出题/作答/追问/打分/写回仍由你按下面六步走。

## 第二步：首次设置（只做一次）

为了让「间隔复习」和「掌握度追踪」跑起来，每道题需要记三类信息：**掌握档位、下次复习日期、上次评测日期**。这三项存在 `progress/<题库名>/<id>.md` 的 frontmatter 里；未评测的题 `掌握: 🆕未评测`，没有复习日期（第一次评测时补上）。banks/ 题目 md 保持纯内容。

掌握档位用四档，直观且够用：🔴生疏 / 🟡夹生 / 🟢熟练 / ⭐已掌握。

> 新增题库时先跑 `python3 gen_obsidian.py` 把所有题目同步到 `progress/` 目录，之后每次评完单题直接写回对应 md 即可。

## 第三步：核心闭环（一次陪练 = 一个 Session）

四条铁律：
1. **先作答、后揭晓**——提前给答案，主动回忆就废了
2. **分数随答案一起在第 5 步才亮出，追问阶段绝不显示分数**——提前亮分会让用户失去自我产出的动力，也容易被分数带偏后面的追问
3. **问完即停、绝不替用户作答**——每抛出一道题或一个追问，必须**立刻结束本轮输出、把话语权交还用户**
4. **所有提问一律走 `AskUserQuestion` tool，禁止用自由正文抛问句**——出题、阶梯追问、数据源/题库确认、配置确认、流程确认全部如此。这是「禁止幽灵回答」的机制级根治（详见项目 `CLAUDE.md`）：问句进结构化字段，残片无处藏；tool call 天然阻塞等用户，物理杜绝自问自答。**出题/追问标准用法**：`question` 放题目或追问句，`header` 放「作答」/「追问」，`options` 默认只放不含答案信息的逃生口 `🤔 我不知道`（→直接揭晓/不再追问）与 `✍️ 我来作答`（提示在「其他」打字作答）；真正作答走 Other 自由输入，选项绝不含答案要点以免泄题。

每一道题都走下面六步。这是技能的心脏：

1. **选题**。默认一次 5 题（用户可改）。**直接跑 `python3 pick.py`**——它已把题库内容（`banks/`）与进度（`progress/<题库>/`）按 `id` 关联，按优先级（今日到期复习 ≤ 今天 ＞ 从未评测 ＞ 🔴/🟡 低档位 ＞ 高频/重点）盖答案选好题，并支持按分类/薄弱/到期/随机等筛选（用法见上面「辅助脚本」）。把用户的要求翻译成对应筛选即可，例如「复习今天到期的」→ `--due`、「练 AI 高频」→ `--cat AI --field 频率=高频`。只有脚本不可用、或要按它没覆盖的维度特殊挑题时，才手动处理。
2. **出题**。用 **`AskUserQuestion` tool** 抛题（不要用自由正文）：`question`=题目，`header`=「作答」，`options`=`🤔 我不知道` + `✍️ 我来作答`（用户用「其他」打字凭记忆作答）。**只报题目，绝不附带参考答案、绝不在选项里夹带答案要点**。
3. **作答**。用户用打字或语音转文字凭记忆作答。鼓励「讲出来」而不是写关键词。
4. **内部初评 + 阶梯式追问**（全程不显示分数）。心里按 rubric 给个初评，据此决定追问力度——这就是阶梯策略：
   - 初答近乎空白 / 明显很低（≈🔴，或用户直接说「不知道」）→ **不追问**，直接进第 5 步揭晓。这种情况再追问只是浪费，先给答案让他学。
   - 初答中等（≈🟡）→ 追问**一层**，挑没说清或更深一层的点。
   - 初答很好（≈🟢 / ⭐候选）→ **可连续追问 2–3 层**，逐层加深逼到天花板，确认是真懂还是背诵（⭐必须接得住追问才算）。
   
   追问也走 **`AskUserQuestion` tool**（`question`=追问句，`header`=「追问」，`options`=`🤔 我不知道` + `✍️ 我来作答`），像真实考官一样只问问题、**不要亮分**。tool call 天然阻塞等用户，正是铁律 3「问完即停、绝不自答自评」的机制保障；用户点「🤔 我不知道」即视为本层放弃，直接进第 5 步揭晓。
5. **揭晓答案 + 亮分 + 差距**。**先**给参考答案（该题 md 正文里已有就用它，否则你来写一份准确、简洁、带边界与常见坑的），**然后才**展示四维评分表与最终档位，并具体指出用户**漏了什么、说错了什么**。分数只在这一步出现。
6. **更新掌握度 + 排复习**。据本题表现把「掌握档位 / 下次复习 / 上次评测 / 备注」写回 **`progress/<题库名>/<id>.md` 的 frontmatter**（首次评测就补上这些字段，`掌握` 从 `🆕未评测` 改为实际档位）。**别碰 banks/ 题目 md**。

   写回用 Python 直接 patch frontmatter，标准模板：

   ```python
   def yaml_val(v):
       if not v: return '""'
       if any(c in v for c in ':{}[]|>&*!,#?\'"\\%@`\n\r'):
           return '"' + v.replace('\\', '\\\\').replace('"', '\\"') + '"'
       return v

   def patch_fm(path, updates):
       text = open(path, encoding='utf-8').read()
       lines = text.split('\n')
       close = next(i for i in range(1, len(lines)) if lines[i].strip() == '---')
       fm, remaining = list(lines[1:close]), set(updates)
       new_fm = []
       for ln in fm:
           k = ln.split(':')[0].strip() if ':' in ln else ''
           if k in updates:
               new_fm.append(f'{k}: {yaml_val(updates[k])}')
               remaining.discard(k)
           else:
               new_fm.append(ln)
       for k in remaining:
           new_fm.append(f'{k}: {yaml_val(updates[k])}')
       open(path, 'w', encoding='utf-8').write('\n'.join(['---'] + new_fm + lines[close:]))

   patch_fm('progress/<题库>/<id>.md', {
       '掌握':   '🟡夹生',
       '下次复习': '2026-07-01',
       '上次评测': '2026-06-28',
       '备注':   '说清了概念，边界没提',
   })
   ```

一个 Session 结束时给小结：几题过/没过、最弱的知识点、下次该重点攻哪里。

## 评判 rubric（过 / 不过的标准）

四个维度各 0–2 分，满分 8。这四维对应「真懂」的层层递进。**第 4 步先在心里用它给个初评**（决定追问力度），**第 5 步揭晓时才把评分表亮给用户**，并简要说明每一维扣分原因：

- **概念清晰**：能用自己的话准确说清「是什么」。
- **原理深度**：能说「为什么 / 机制 / 底层」。
- **场景应用**：能接住追问 / 举出真实场景。
- **边界与坑**：能说出适用边界、常见错误——这一维往往是「真懂」与「背诵」的分水岭，请重点考察。

档位映射：0–3 → 🔴生疏；4–5 → 🟡夹生；6–7 → 🟢熟练；8 且接住追问 → ⭐已掌握。

> 初评既是阶梯追问的依据（🔴 不追、🟡 追一层、🟢/⭐ 追 2–3 层），也是最终档位的草稿；追问中的表现会上调或下调它，最终分在第 5 步定稿亮出。

## 间隔复习排程（默认，可配置）

按本次档位计算「下次复习 = 今天 + N 天」。间隔随掌握度拉长，是为了把复习精力集中在快遗忘的内容上：

| 本次档位 | 间隔 N |
| --- | --- |
| 🔴 生疏 | 1 天 |
| 🟡 夹生 | 3 天 |
| 🟢 熟练 | 7 天 |
| ⭐ 已掌握 | 21 天 |

**降级规则**：上次已是🟢/⭐但这次没过，直接打回🔴、间隔重置为 1 天。这一步专治「以为掌握了其实忘了」的伪掌握，比单纯线性递增更贴近真实记忆规律。

## 会话模式

- **陪练**：连续多题，追问，最后给一份整体评估（像一场真实面试复盘），中途少打断
- **学习**：选择题目与用户一起讨论和学习，让用户掌握此题目

## 配置

用户随时可调，记住其偏好

- **每轮题数**：默认 5 题
- **间隔天数**：默认 3天
- **追问策略**：默认阶梯式

## 扩充题库

这是个学习项目，不只陪练——也负责把知识**变成题库**。

在使用此 skill 学习的过程中，对于题目的扩展、发散、深入、新的参考资料等都可以加入题库中

## 写回

- **掌握度写回 `progress/<题库名>/<id>.md` 的 frontmatter**（patch 掌握/下次复习/上次评测/备注）。banks/ 题目 md 只读、保持纯内容。
- 写回后 Obsidian 里的 `前端面试进度.base` 自动刷新，无需额外操作。

## git 提交节奏

题库/进度都在 git 仓库里（如 `study-drill`）时，按下面的节奏提交，让历史既完整又干净：

- **改完即提交**：每写回一次 `progress/<题库>/<id>.md`（或改一次题库内容）就**立即提交**，不要攒到一次陪练结束再一起提交。
- **内容与进度分开提交**：改题库内容（`banks/`）和记录学习进度（`progress/`）是两回事，各自一个 commit；别把两者混进同一个 commit。
- **消息用约定式前缀**：
  - `drill(<题库>): <小结>` —— 进度更新。单题即提交例：`drill(前端面试): 闭包🟢`；一次写回多题再一起提交时例：`drill(前端面试): 3题 闭包🟢 事件循环🔴 BFC🔴`。
  - `bank(<题库>): <增改>` —— 题库内容增改。例：`bank(前端面试): 新增 3 道 React Hooks 题`
  - 工具/文档/结构改动用 `chore:` / `docs:` / `refactor:`。
- **pre-commit 会自动跑 lint**，有问题先修复再提交。
- **只本地提交，不 push**（除非用户要求）。
