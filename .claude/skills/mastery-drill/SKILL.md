# 掌握度陪练（Mastery Drill）

## 这个技能要解决的核心问题

学习者最大的盲区是「能力自评失真」：看过答案、标记「已读/已背」就以为掌握了，但一被追问就露馅。真正的掌握不是「看过」，而是**能脱稿讲清「是什么/为什么/什么场景/边界与坑」，并接得住追问**。

本技能把抽象的「我掌握了吗」变成一个可量化、可定位、可持续的训练闭环。核心是三件被学习科学反复验证的事：**主动回忆**（先盖答案自己产出，而不是重读）、**针对性追问**（暴露伪掌握）、**间隔复习**（对抗遗忘曲线）。请始终围绕这三点设计每一次互动。

## 第一步：确定数据源与跟踪方式

不同用户的材料存放方式不同，先判断用哪种，再开始。**不要假设，先快速确认。**

1. **本地题库（默认，纯本地，题库与进度分离）**：题目内容和学习进度分开存——
   - **题目内容**：`banks/<题库名>/<分类>/<题目>.md`，frontmatter **只存内容**（题目/分类/id + 该题库 `meta.yml` 声明的可选字段，如 频率），正文存答案。每个题库一个 `meta.yml` 声明分类清单与可选字段。
   - **学习进度**：`progress/<题库名>.tsv`（Tab 分隔，表头 `id 掌握 下次复习 上次评测 备注`），**只有评过的题才有行**，按 `id` 关联回题目。

   ```markdown
   # banks/前端面试/JS/深拷贝.md —— 纯内容，绝不含进度字段
   ---
   题目: 深拷贝
   分类: JS
   频率: 高频
   id: 55e72a4d-...
   ---
   深拷贝递归复制对象所有层级……（答案正文）
   ```
   ```
   # progress/前端面试.tsv —— 列间是 Tab
   id    掌握    下次复习    上次评测    备注
   55e72a4d-...    🔴生疏    2026-06-25    2026-06-24    循环引用(WeakMap)忘了…
   ```

   陪练：选题读「内容 + 进度」按 `id` 关联、揭晓读正文、评完**只写 `progress/<题库>.tsv`**，不依赖任何外部服务。
2. **用户提供的题目文件/列表**（CSV、Markdown、文档、或直接粘贴）：按上面的字段在内存里跟踪，或落成同样的本地 md。
3. **只给了一个主题**（如「考考我 React Hooks」「帮我复习操作系统进程调度」）：现场按主题生成题目来问，并落成本地 md 累积，这样下次能接着练。

确认时一句话即可，例如：「练哪一份？本地题库目录、给我一份题目文件、还是报个主题我来出题？」

## 第二步：首次设置（只做一次）

为了让「间隔复习」和「掌握度追踪」跑起来，每道题需要记三类信息：**掌握档位、下次复习日期、上次评测日期**。这三项放在 `progress/<题库名>.tsv` 里，按 `id` 关联回题目；未评测的题在 TSV 里没有行（第一次评测时新增）。题库 md 保持纯内容。

掌握档位用四档，直观且够用：🔴生疏 / 🟡夹生 / 🟢熟练 / ⭐已掌握。（想要更细可改成 0–100 的分数，但默认四档即可，别过度设计。）

## 第三步：核心闭环（一次陪练 = 一个 Session）

每一道题都走下面六步。这是技能的心脏，两条铁律：① **先作答、后揭晓**——提前给答案，主动回忆就废了；② **分数随答案一起在第 5 步才亮出，追问阶段绝不显示分数**——提前亮分会让用户失去自我产出的动力，也容易被分数带偏后面的追问。

1. **选题**。默认一次 5 题（用户可改）。**把题库内容（`banks/`）与进度（`progress/<题库>.tsv`）按 `id` 关联起来选题**。优先级：今日到期复习（下次复习 ≤ 今天）＞ 从未评测（进度里没行）＞ 🔴/🟡 低档位 ＞ 高频/重点。支持按分类、按薄弱、按到期、随机等筛选。
2. **出题**。只报题目，**绝不附带参考答案**。一次问一道，等用户作答再继续（不要一次性把 5 题都抛出来）。
3. **作答**。用户用打字或语音转文字凭记忆作答。鼓励「讲出来」而不是写关键词。
4. **内部初评 + 阶梯式追问**（全程不显示分数）。心里按 rubric 给个初评，据此决定追问力度——这就是阶梯策略：
   - 初答近乎空白 / 明显很低（≈🔴，或用户直接说「不知道」）→ **不追问**，直接进第 5 步揭晓。这种情况再追问只是浪费，先给答案让他学。
   - 初答中等（≈🟡）→ 追问**一层**，挑没说清或更深一层的点。
   - 初答很好（≈🟢 / ⭐候选）→ **可连续追问 2–3 层**，逐层加深逼到天花板，确认是真懂还是背诵（⭐必须接得住追问才算）。
   
   追问像真实考官一样只问问题，**不要亮分**。
5. **揭晓答案 + 亮分 + 差距**。**先**给参考答案（该题 md 正文里已有就用它，否则你来写一份准确、简洁、带边界与常见坑的），**然后才**展示四维评分表与最终档位，并具体指出用户**漏了什么、说错了什么**。分数只在这一步出现。
6. **更新掌握度 + 排复习**。据本题表现把「掌握档位 / 下次复习 / 上次评测 / 备注」写回 **`progress/<题库名>.tsv` 里该 `id` 的行**（首次评测就新增一行）。**别碰题库 md**。

一个 Session 结束时给小结：几题过/没过、最弱的知识点、下次该重点攻哪里。

## 评判 rubric（过 / 不过的标准）

四个维度各 0–2 分，满分 8。这四维对应「真懂」的层层递进。**第 4 步先在心里用它给个初评**（决定追问力度），**第 5 步揭晓时才把评分表亮给用户**，并简要说明每一维扣分原因：

- **概念清晰**：能用自己的话准确说清「是什么」。
- **原理深度**：能说「为什么 / 机制 / 底层」。
- **场景应用**：能接住追问 / 举出真实场景。
- **边界与坑**：能说出适用边界、常见错误——这一维往往是「真懂」与「背诵」的分水岭，请重点考察。

档位映射：0–3 → 🔴生疏；4–5 → 🟡夹生；6–7 → 🟢熟练；8 且接住追问 → ⭐已掌握。

> 初评既是阶梯追问的依据（🔴 不追、🟡 追一层、🟢/⭐ 追 2–3 层），也是最终档位的草稿；追问中的表现会上调或下调它，最终分在第 5 步定稿亮出。

## 间隔复习排程（默认，可配置）

按本次档位计算「下次复习 = 今天 + N 天」。间隔随掌握度拉长，是为了把复习精力集中在快遗忘的内容上：

| 本次档位 | 间隔 N |
| --- | --- |
| 🔴 生疏 | 1 天 |
| 🟡 夹生 | 3 天 |
| 🟢 熟练 | 7 天 |
| ⭐ 已掌握 | 21 天 |

**降级规则**：上次已是🟢/⭐但这次没过，直接打回🔴、间隔重置为 1 天。这一步专治「以为掌握了其实忘了」的伪掌握，比单纯线性递增更贴近真实记忆规律。

## 会话模式

支持几种触发后可直接进入的模式，按用户措辞自动选择：

- **到期复习**（默认日常）：只练今天到期的题。
- **定向**：按分类/主题/某家公司练（「只练 AI 类」「考考我闭包相关」）。
- **薄弱强化**：专挑🔴/🟡和最近没过的题。
- **模拟面试/模拟考**：连续多题、追问更狠、最后给一份整体评估（像一场真实面试复盘），中途少打断。

## 写回与配置

- **掌握度写回 `progress/<题库名>.tsv`**（按 `id` 改/增行：掌握/下次复习/上次评测/备注）。题库 md 只读、保持纯内容。
- 可配置项（用户随时可调，记住其偏好）：每轮题数、间隔天数表、**追问策略**（默认阶梯式；可改成「每题固定追问一层」或「从不追问」）、语言（中/英，可用于模拟英文面试）。

## git 提交节奏（题库在 git 仓库时）

题库/进度都在 git 仓库里（如 `study-drill`）时，按下面的节奏提交，让历史既完整又干净：

- **一次陪练 = 一个 commit**：在给完 Session 小结后，把本次进度改动**一次性提交**，别每题一个 commit。
- **内容与进度分开提交**：改题库内容（`banks/`）和记录学习进度（`progress/`）是两回事，各自一个 commit；一次会话里若既加题又练题，就两个 commit。
- **消息用约定式前缀**：
  - `drill(<题库>): <小结>` —— 进度更新。例：`drill(前端面试): 5题 深拷贝🔴 Transformer🔴 向量库🔴 虚拟列表🟡 FC/MCP🟡`
  - `bank(<题库>): <增改>` —— 题库内容增改。例：`bank(前端面试): 新增 3 道 React Hooks 题`
  - 工具/文档/结构改动用 `chore:` / `docs:` / `refactor:`。
- **pre-commit 会自动跑 lint**，不过就拦下——按提示先修正（进度字段误写进题库、孤儿 id、日期非法等）再提交。
- **只本地提交，不 push**（除非用户要求）。

## 刻意不做的（保持轻量）

不做实时面试「作弊」辅助（只做练习与能力建设）、不自建独立应用、不强行量化主观表达。本技能只验证并强化一件事：**主动回忆 + 追问评判 + 间隔复习**能否真正提升掌握度与信心。

## 一个完整示例（节选）

**Input（用户）**：开始陪练，今天到期的

**Output（技能行为）**：
1. 扫题库目录各题 frontmatter，选出今日到期 5 题（`下次复习 ≤ 今天`），先问第 1 题：「SSE 与 WebSocket 在大模型对话场景如何选型？」（不给答案、不给分）
2. 用户作答后，心里初评 ≈🟢 → 不亮分，追问一层：「HTTP/1.1 下同源 6 连接限制怎么破？」
3. 用户答得不错够到 🟢 → 再追一层逼天花板：「HTTP/2 之后还需要担心连接数吗？SSE 断线怎么自动重连？」（仍不亮分）
4. 揭晓：**先**给参考答案，**再**亮评分表（概念2/原理2/场景1/边界1=6→🟢）+ 差距：「你没提 `Last-Event-ID` 自动续传，这是 SSE 续传的关键」。
5. 写回本地文件：档位=🟢熟练、下次复习=今天+7 天、上次评测=今天。
6. 轮到用户直接说「不知道」的题：**跳过追问**，直接揭晓答案、记 🔴、明天再考。
7. 全部做完给小结：「5 题里 2 题 🟢、1 题 🟡、2 题 🔴，最弱是『流式中断恢复』，明天它会再出现。」
