# study-drill 项目纪律

## 机制护栏：所有提问一律走 AskUserQuestion tool（根治幽灵回答）

陪练时**任何向用户提问的动作**——出题、阶梯追问、选数据源/题库、配置确认（每轮题数/间隔/策略）、流程确认（是否进下一题）——**一律用 `AskUserQuestion` tool，禁止用自由正文抛问句**。

这是幽灵回答的**机制级根治**，优先于下面的自律铁律。原理：

- **残片无处藏**：问句进的是 tool 的结构化字段，不再经过自由正文流，「思考区 token 滑进正文」这种瞬时滑手失去了滋生环境。
- **物理杜绝自问自答**：tool call 天然阻塞、必须等用户返回结果，从机制上断绝「自答自评自分」（对应下方铁律 2）。

**出题 / 追问的标准用法**：`question` 放题目或追问句；`header` 放「作答」或「追问」；`options` 默认只放**不含任何答案信息**的逃生口——`🤔 我不知道`（→ 直接揭晓 / 不再追问）与 `✍️ 我来作答`（提示用户在「其他」输入框打字作答）。**真正的作答走 Other 自由输入，所以选项里绝不能出现答案要点**，否则等于泄题、破坏主动回忆。用户点「我不知道」＝放弃本层，按铁律进入揭晓。

# 题库

- 题库目录为 `banks/[题库]/[分类]/[题目].md`
- 一个题目包含如下部分
	- 题干
	- 元数据
	- 答案：按照 **金字塔原理** 结构化，分层编写，方便记忆
	- 追问：由题目引申和发散出的，更深层的原理和细节等
	- 参考资料：权威资料的引用
