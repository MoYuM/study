流程：解析 HTML 生成 DOM 树、解析 CSS 生成 CSSOM → 合成渲染树（Render Tree）→ 布局 Layout（计算几何位置）→ 分层 → 绘制 Paint → 分块/光栅化（GPU 栅格化）→ 合成 Composite 显示。

- 回流（Reflow）：改变几何属性（尺寸/位置）触发 Layout，开销大。
- 重绘（Repaint）：改变样式但不影响布局（如颜色）。
- `transform` / `opacity` 走合成层，不触发回流重绘，性能最好。
- 优化：减少逐项操作 DOM（用 DocumentFragment / 批量）、避免强制同步布局（读写分离）、用 class 批量改样式。
