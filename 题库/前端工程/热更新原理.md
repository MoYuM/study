HMR（热模块替换）：开发时模块代码变更后，通过 WebSocket 通知浏览器，运行时只替换变更的模块，不刷新整页、保留应用状态。

- Webpack-dev-server：监听文件变化 → 编译出补丁（manifest + 更新 chunk）→ 通过 WS 推送 → 客户端 runtime 用 `module.hot.accept` 替换模块。
- Vite：基于浏览器原生 ESM，仅让浏览器重新请求变更的模块，无需打包，更新更细、速度更快。
