`0.1 + 0.2 !== 0.3`：因为 IEEE 754 双精度浮点用二进制表示，十进制小数 0.1/0.2 无法精确表示，运算后产生误差（0.30000000000000004）。

解决：

- 用 `Number.EPSILON` 做容差比较：`Math.abs(a-b) < Number.EPSILON`。
- 放大为整数运算再缩小（乘 10^n）。
- `toFixed(n)` 控制精度后再比较。
- 高精度场景用 decimal.js / big.js。
