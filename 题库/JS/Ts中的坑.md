常见坑：

- any 会「传染」并绕过类型检查；尽量用 unknown + 收窄。
- 类型断言 as 只骗编译器，运行时不转换，断言错会埋雷。
- enum 有运行时产物且数字枚举可反向映射；常用 const enum 或联合字面量替代。
- interface 与 type 区别（声明合并、扩展方式）。
- 协变/逆变导致的函数参数类型不兼容。
- 严格模式下 null/undefined 检查、可选链与非空断言滥用。
- 第三方库缺 d.ts、泛型默认值与约束。
